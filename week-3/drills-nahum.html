Resources: w3schools - Function Invocations
  1. function multiply1(x, y) { return x * y; } function multiply2(x, y) { x * y; }
    • What is the result of multiply1(5, 2) === 10? multiply2(5, 2) === 10?

      • multiply1(5,2) === 10; //-> true
      multiply2(5,2) === 10; //-> false

      This is because: The first function returns the value of (x * y) and that returned number is what is being set strictly equal to 10; the second function does not return any value (even though it does multiply the inputs) and therefore is setting (undefined) strictly equal to 10 which returns false.
    • What is the result of multiply1(5, 2) !== 10? multiply2(5, 2) !== 10?

      • multiply1(5, 2) !== 10; //-> false multiply2(5, 2) !== 10; //-> true

      This is because: (a1) the function multiply1(5, 2); returns 10, and 10 is strictly equivalent to 10.
      (a2) because the multiply2() function does not return a value it will always return undefined, undefined is !== 10, and therefore the expression is true.
    • What is the result of typeof multiply1(5, 2)? typeof multiply2(5, 2)?

      • typeof multiply1(5, 2); //-> "number".
      typeof multiply2(5, 2); //-> "undefined".
    • What is the result of isNaN(multiply1(5, 2))? isNaN(multiply2(5, 2))?

      • isNaN multiply1((5, 2)); //-> "false".
      isNaN multiply2((5, 2)); //-> "true".
    • What is the result of !isNaN(multiply1(5, 2))? !isNaN(multiply2(5, 2))?

      • !isNaN multiply1((5, 2)); //-> "true".
      !isNan multiply2((5, 2)); //-> "false".
    • What is the result of multiply1(5, 2) + multiply1(100,2)? multiply2(5, 2) + multiply2(100,2)?

      • multiply1(5, 2) + multiply1(100,2); //-> 210.
      multiply2(5, 2) + multiply2(100,2); //-> NaN.
    • What is the result of multiply1 + multiply2(5, 1)?

      • multiply1 + multiply2(5, 1); //-> NaN
    • Create a statement using only the function multiply1 and + to compute 997

      • multiply1(1, 1) + multiply1(498, 2); //-> 997
  2. var myObject = { firstName:"John", lastName: "Doe", fullName: function() { return this.firstName + " " + this.lastName; }, myself: function() { return this; } };
    • What is the result of myObject.firstName? myObject.lastName?

      • myObject.firstName; //-> "John" myObject.lastName; //-> "Doe"

    • What is the result of myObject.fullName?

      • myObject.fullName; //-> function () { return this.firstName + " " this.lastName; } without adding the parentheses calling the myObject.fullName function, it will only return the value of the key: fullName, which is equal to the function (object) not the value of the function being called.

    • What is the result of myObject.fullName()?

      • myObject.fullName(); //-> "John Doe"

    • What is the result of myObject.fullName('Jane', 'Smith')?

      • myObject.fullName('Jane', 'Smith'); //-> "John Doe"
      This is because the function object (the value of) to fullName takes no parameters/arguments, so adding them as part of the function call does not change the output of the original function (which is "John Doe")

    • What is the result of myObject.myself?

      • myObject.myself; //-> function () { return this; }

      without adding the parentheses calling the myObject.myself function, it will only return the value of the key: myself, which is equal to the function (object) not the value of the function being called.

    • What is the result of myObject.myself()?

      • myObject.myself; //-> Object {firstName: "John", lastName: "Doe"}

      BY adding the parentheses you're calling the function myself() which returns the value of running the function itself which is this. this will return the Object myObject, which lists all the key/value pairs that make up myObject.

    • What is the result of myObject.myself(23)?

      • myObject.myself(23); //-> Object {firstName: "John", lastName: "Doe"} see above for why adding arguments (23) does nothing to affect the output.

Scopes & Variable Hoisting

If you get any of the following wrong after double-checking your answers in the console, review the following resource: Everything you wanted to know about javascript scope
  1. function foo() { var x = 3; console.log(x); } console.log(x);
    Run the above code. Explain what happened. ANSWER: variables declared within a function are only accessible within that function's scope.
    This means that if we try to log or use the variable 'x' we will be returned a reference error
  2. var greeting = 'Hello'; var scope1 = function () { var scope2 = function () { var scope3 = function () { console.log(greeting); }; scope3(); }; scope2(); };
    What is the result of scope1()?
    ANS: calling scope1 will log 'hello' in the console. Because greeting is declared from outside the function , it is available to each of the scope functions.
  3. var greeting = 'Hello'; var scope1 = function () { var scope2 = function () { var scope3 = function () { return greeting; }; return scope3(); }; return scope2(); };
    What is the result of scope2()?

    scope2(); //-> VM903:1 Uncaught ReferenceError: scope2 is not defined(…)(anonymous function) @ VM903:1
    This is because the scope2() declaration is nested within the scope1() declaration, therefore it is unavailable to the outer context.

  4. var greeting = 'Hello'; var scope1 = function () { var greeting = 'Goodbye'; var scope2 = function () { var scope3 = function () { return greeting; }; return scope3(); }; return scope2(); };
    What is the result of scope3()?

    scope3(); //-> VM904:1 Uncaught ReferenceError: scope2 is not defined(…)(anonymous function) @ VM904:1
    This is because the scope3() declaration is nested within the scope1() declaration, therefore it is unavailable to the outer context.
